SpringBoot环境搭建

SpringBoot简介

  • SpringBoot可以用来简化Spring+SpringMVC+Mybatis框架的搭建
  • SpringBoot不需要进行复杂的xml配置，采用大量的默认配置来简化噢诶之过程

SpringBoot特性

  • 能够快速创建基于Spring的新程序
  • 能够直接使用java main方法启动内嵌的Tomcat，不需要部署war文件
  • 提供约定的starter POM来简化MAVEN配置
  • 根据MAVEN配置自动搭建Spring SpringMVC等
  • 提供程序的健康检查
  • 采用注解配置

SpringBoot四大核心

  • 自动配置：针对很多Spring常见应用，SpringBoot能够自动配置
  • 起步依赖：告诉SpringBoot需要什么功能，他就能引入需要的依赖库
  • Actuator: 健康检查
  • 命令行界面：可选特性

SpringBoot环境搭建

  • 打开eclipse，进入Help->Eclipse Marketplace
  • 搜索STS（Spring创建工具）,安装
  • 打开Window->Preference->Maven->setting,设置MAVEN
  • 新建一个Spring Starter Project
  • 填写项目名称，下一步
  • 勾选WEB 完成

1.3 线程生命周期
状态 说明 状态转换
NEW 创建一个Thread对象，不处于执行状态 通过start进入RUNNABLE状态
RUNNABLE 创建线程，等待CPU调度 CPU调度进入RUNNING状态
RUNNING 执行状态 * 调用stop或判断标识符进入TERMINATED状态
RUNNING 执行状态 * 调用sleep或wait进入BLOCKED状态
RUNNING 执行状态 * 进行某个阻塞的IO操作，进入BLOCKED状态
RUNNING 执行状态 *获取某个锁资源，从而加入到该锁的阻塞队列而进入BLOCKED状态
RUNNING 执行状态 * 由于CPU调度进入RUNNABLE状态
RUNNING 执行状态 * 线程主动调用yield方法进入RUNNABLE状态
BLOCKED 阻塞状态 * 直接进入TERMINATED状态，调用stop或意外错误
BLOCKED 阻塞状态 * 线程阻塞的操作结束，进入RUNNABLE状态
BLOCKED 阻塞状态 * 线程完成了指定时间的休眠，进入RUNNABLE状态
BLOCKED 阻塞状态 * Wait被其他线程notify/notifyall打断
BLOCKED 阻塞状态 * 线程获取到了某个锁资源，进入RUNNABLE状态
BLOCKED 阻塞状态 * 线程在阻塞过程被打断，进入RUNNABLE状态
TERMINATED 最终态

第二节：线程的父子关系

任何一个线程都是由他的父线程创建的，上述示例中所有线程都是由main作为父线程
currentThread():查看当前线程

第三节：Thread与ThreadGroup

  • main线程所在的ThreadGroup成为main
  • 构造一个线程的时候如果没有显式的指定ThreadGroup，那么他将和父线程在同一个Group中，拥有相同的优先级

第四节：Thread与RUNNABLE

Runnable接口非常简单，只定义了一个无参数无返回的run方法
创建线程的两种方式：继承Thread和实现Runnable，其实实现Runnable实现主要还是Thread类来执行，只是通过Runnbale来实现自己的业务逻辑
