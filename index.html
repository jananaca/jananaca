使用场景：明确的计划不同条件下创建不同的实例。

Thread API

线程sleep

sleep方法重载

  • public static void sleep(long millis) throws InterruptedException//传入毫秒数
  • public static void sleep(long millis，int nanos) throws InterruptedException//传入毫秒数和纳秒数

    sleep方法介绍

    sleep方法会使当前线程进入指定毫秒数的休眠，暂停执行，虽然给定了一个休眠的时间，但是最终要以系统的定时器和调度器的精度为准。休眠有一个重要特性，那就是其不会放弃monitor锁的所有权。
    注：monitor锁：监视锁，每一个对象都具有唯一的一个监视锁，同时只有一个线程可以拥有一个对象的监视锁。一个线程通过调用某个对象的wait方法来释放monitor进入休眠状态。
    获取监视锁的三种方法
  • public synchronize a () {} 执行该对象的同步方法
  • synchronize(obj) {} 执行该对象的同步块
  • public static synchronize b(){} 执行某个类的同步方法
    sleep只执行当前线程的休眠，互不影响

    TimeUnit替代Thread.sleep()

    优势：省去了时间的换算步骤
    例：TimeUnit.HOURS.sleep(3):休眠三小时

    线程yield

    yield方法介绍

    yield方法属于启发式的方法，会提醒调度器愿意放弃当前CPU资源。如果CPU不紧张，则会忽略这个提醒。
    就比如和调度器说你先忙，我不着急，等你忙完了再帮我。
    注：调用yield会使线程从RUNNING状态切换到RUNNABLE状态，而sleep则会使线程短暂的进入block状态。

    设置线程的优先级

    相关函数

  • public final void setPriority(int newPriority)//设置优先级
  • public final int getPriority()//获取优先级

    优先级介绍

  • 优先级越大，优先级越高
  • 当CPU比较繁忙的时候，高的优先级可能会获得更多执行的机会，否则几乎没有区别。

    优先级源码分析

有源码可知，线程的最大优先级为当前线程组所允许的最大优先级，如果超过了则会失效取代为当前线程组的最大优先级。

SpringBoot环境搭建

SpringBoot简介

  • SpringBoot可以用来简化Spring+SpringMVC+Mybatis框架的搭建
  • SpringBoot不需要进行复杂的xml配置，采用大量的默认配置来简化噢诶之过程

SpringBoot特性

  • 能够快速创建基于Spring的新程序
  • 能够直接使用java main方法启动内嵌的Tomcat，不需要部署war文件
  • 提供约定的starter POM来简化MAVEN配置
  • 根据MAVEN配置自动搭建Spring SpringMVC等
  • 提供程序的健康检查
  • 采用注解配置

SpringBoot四大核心

  • 自动配置：针对很多Spring常见应用，SpringBoot能够自动配置
  • 起步依赖：告诉SpringBoot需要什么功能，他就能引入需要的依赖库
  • Actuator: 健康检查
  • 命令行界面：可选特性

SpringBoot环境搭建

  • 打开eclipse，进入Help->Eclipse Marketplace
  • 搜索STS（Spring创建工具）,安装
  • 打开Window->Preference->Maven->setting,设置MAVEN
  • 新建一个Spring Starter Project
  • 填写项目名称，下一步
  • 勾选WEB 完成
