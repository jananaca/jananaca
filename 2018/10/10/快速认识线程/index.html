1.3 线程生命周期
状态 说明 状态转换
NEW 创建一个Thread对象，不处于执行状态 通过start进入RUNNABLE状态
RUNNABLE 创建线程，等待CPU调度 CPU调度进入RUNNING状态
RUNNING 执行状态 * 调用stop或判断标识符进入TERMINATED状态
RUNNING 执行状态 * 调用sleep或wait进入BLOCKED状态
RUNNING 执行状态 * 进行某个阻塞的IO操作，进入BLOCKED状态
RUNNING 执行状态 *获取某个锁资源，从而加入到该锁的阻塞队列而进入BLOCKED状态
RUNNING 执行状态 * 由于CPU调度进入RUNNABLE状态
RUNNING 执行状态 * 线程主动调用yield方法进入RUNNABLE状态
BLOCKED 阻塞状态 * 直接进入TERMINATED状态，调用stop或意外错误
BLOCKED 阻塞状态 * 线程阻塞的操作结束，进入RUNNABLE状态
BLOCKED 阻塞状态 * 线程完成了指定时间的休眠，进入RUNNABLE状态
BLOCKED 阻塞状态 * Wait被其他线程notify/notifyall打断
BLOCKED 阻塞状态 * 线程获取到了某个锁资源，进入RUNNABLE状态
BLOCKED 阻塞状态 * 线程在阻塞过程被打断，进入RUNNABLE状态
TERMINATED 最终态
